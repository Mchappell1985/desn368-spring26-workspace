CSS Resets Research Notes

Web Design Research Mini Site

Step 1: Historical Foundation

Summary Notes

The creation of CSS resets was heavily attributed to Eric Meyer. At the time, browsers used different default stylizations for their elements. The inconsistent display of websites had developers wasting time making browser-specific tweaks instead of creating layouts that applied to all browser types.

Meyers is the reason why developers are now able to create CSS styles from scratch, without having to worry about whatever browser the viewer is using to interfere.

Things to Notice

  • In 2007, there wasn't a set standard for browsers before this was used.
  • Consistency was top priority for most developers.
  • Resets were the blank slate that developers had wanted.

Step 2: Modern Evolution

Summary Notes

Josh Comeau brings up the point that modern browsers retain consistency, so the CSS resets that developers had in the past aren't necessarily needed.

The way that he resets isn't a complete wipe. It tackles the problems/behaviors that are most common in sites. Adjustments to aspects that need improvement, but now entirely cleaning the slate like before.

Josh also prioritizes accessibility while tying it into the modernized development landscape.

Things that have changed since 2007

  • Browsers became more standardized
  • Accessibility matters more
  • Resets are more selective/specific, and it tends to assist with extra work on a developers end.

Step 3: Wider Landscape

Jeff Star decided to provide an example of how developers uniquely approach CSS resets by collecting different reset styles and comparing them.

When comparing these different styles, he noticed that some resets perserved a lot of styling that was seen as useful to them, while others were hardcore resets that took away almost every broswer default.

So while reset preferences are technically up to the developer, and lot of individuals choose their resets based odd of what project they're creating.

Things to Notice

  • Resets vary depending on the developer.
    Some developers perfer control over their site in comparison to convenience.
  • Developers tend to customize what reset they're planning to utilize based off of the task.

Step 4: Critical Questions

Why reset at all?

CSS resetting removes inconsistent browser styles, so websites look similar across all different browser types.

Why Josh's version over Eric's?

  • Keeps browser defaults that still benefit the developer.
  • Assists with accessability features and improves them.
  • Uses box-sizing rules, which is universal.

How do you use resets?

Add them with a link tag, import statement, or directly inside a CSS file. 

 
        
< link rel="stylesheet" href="reset.css">

        or

@import url("reset.css");

What is a CSS boilerplate?

A starter set of code that includes all of the resets and deafult styles. Normally used at the beginning of a project.

Why use a boilerplate?

Helps gives you a starting point for your code.

Why is Normalize.css different?

Normalize.css utilizes the capability to preserves useful browser styling instead of deleting everything. It is an improvement on what already exists instead.
